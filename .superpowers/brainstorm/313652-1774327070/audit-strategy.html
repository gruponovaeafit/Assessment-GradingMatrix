Strategy Comparison

How should we implement the logging for these priority areas?

1

Explicit Manual Logging

Manually call logAudit() inside each API route. Gives full control over the "Detalles" string for each specific action.

  • Precise messages
  • Simple to understand
  • Boilerplate code
  • Easy to forget one
2

Generic Logging Middleware

Wrap API handlers in a withAudit middleware that automatically logs the request (method, path, user) after it completes.

  • Automatic coverage
  • Less code repetition
  • Harder to log details
  • Noisy logs
3

Hybrid Approach (Recommended)

Use manual logging for high-value actions (Staff, Assessment) and a standardized helper for common operations.

  • Best of both worlds
  • High signal-to-noise
  • Slightly more complex
